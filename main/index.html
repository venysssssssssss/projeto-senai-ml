    Aqui está a documentação em formato Markdown para o script principal do projeto:

    Script Principal para Execução do Projeto

    Este script orquestra a execução completa do projeto, desde a geração dos datasets, treinamento do modelo, avaliação, até a visualização dos resultados. O script é configurado para ser executado como um ponto de entrada principal para o pipeline de machine learning.

    Dependências

    • sys e os: Usados para manipulação de caminhos e configuração do ambiente de execução.
    • logging: Para registro de logs durante a execução do script.
    • data_generation: Módulo responsável pela geração e salvamento dos datasets.
    • model_first: Módulo responsável pelo carregamento, pré-processamento dos dados, e treinamento do modelo.
    • evaluation: Módulo responsável pela avaliação do modelo treinado.
    • model_visualization: Módulo para visualização do desempenho do modelo.

    Função Principal

    main

    def main():
    # Carregar configurações
    config = carregar_config("config/config.yaml")

    # Configurar logging
    logging.basicConfig(filename=config['logging']['log_file'], level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s')

    # Geração dos datasets
    logging.info("Gerando datasets de treino e teste")
    dataset_treino = gerar_dataset(config['data']['num_samples'])
    dataset_teste = gerar_dataset(int(config['data']['num_samples'] * config['data']['test_size']))
    salvar_dataset(dataset_treino, "data/dataset_treino.csv")
    salvar_dataset(dataset_teste, "data/dataset_teste.csv")

    # Treinamento do modelo
    logging.info("Treinando o modelo")
    dados_treino = carregar_dados("data/dataset_treino.csv")
    X_treino, y_treino, preprocessor = preprocessar_dados(dados_treino)
    feature_names = obter_nomes_features(preprocessor, X_treino)
    modelo = treinar_modelo(X_treino, y_treino, preprocessor, config['model']['save_path'])

    # Avaliação do modelo
    logging.info("Avaliando o modelo treinado")
    dados_teste = carregar_dados("data/dataset_teste.csv")
    X_teste, y_teste, preprocessor = preprocessar_dados(dados_teste)
    modelo = carregar_modelo(config['model']['save_path'])
    metrics = avaliar_modelo(modelo, X_teste, y_teste)

    logging.info("Métricas de Avaliação do Modelo:")
    for metric, value in metrics.items():
        logging.info(f"{metric}: {value:.4f}")

    # Visualização dos resultados
    y_pred = modelo.predict(X_teste)
    plot_predictions(y_teste, y_pred)
    plot_residuals(y_teste, y_pred)
    plot_feature_importances(modelo, feature_names)

if __name__ == "__main__":
    main()

    Descrição

    A função main() coordena as etapas principais do projeto:

    1. Carregar Configurações: As configurações são carregadas de um arquivo YAML, especificando parâmetros como número de amostras, tamanho do conjunto de teste, e caminhos para salvar modelos e logs.

    2. Configuração de Logging: Define o formato e o arquivo para o log, registrando as etapas da execução do pipeline.

    3. Geração dos Datasets: Cria datasets de treino e teste usando a função gerar_dataset e salva os arquivos CSV resultantes.

    4. Treinamento do Modelo: Carrega os dados de treino, realiza o pré-processamento e treina um modelo de árvore de decisão. O modelo treinado é salvo para uso posterior.

    5. Avaliação do Modelo: Avalia o modelo treinado usando o conjunto de teste e calcula métricas de desempenho como MAE, MSE, e R². Essas métricas são registradas no log.

    6. Visualização dos Resultados: Gera gráficos para visualizar a relação entre valores reais e preditos, distribuição dos resíduos, e importância das features.

    Execução

    Este script deve ser executado como o ponto de entrada principal do projeto:

    python main.py

    Ao executá-lo, o pipeline completo de machine learning será executado, desde a geração dos dados até a visualização dos resultados.

    Logs

    Os logs da execução serão salvos no caminho especificado no arquivo config.yaml, permitindo o rastreamento detalhado de todas as etapas.

    Considerações

    • Modularidade: O script utiliza funções de módulos separados, facilitando a manutenção e a reutilização de código.
    • Flexibilidade: O uso de um arquivo de configuração YAML permite ajustar parâmetros do pipeline sem modificar o código-fonte.
    • Visualização: A visualização das previsões e dos resíduos fornece uma compreensão intuitiva do desempenho do modelo.
